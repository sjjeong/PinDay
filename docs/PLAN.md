# PinDay 앱 기획서

## 앱 소개

**PinDay**는 소중한 기념일을 핀(Pin)처럼 꽂아두고 한눈에 확인하는 앱입니다.
특히 음력 생일처럼 매년 날짜가 바뀌는 기념일도 자동으로 양력 변환하여 놓치지 않도록 도와줍니다.

## 만들게 된 배경

- 어른들의 생일이 음력 기준이라 매년 양력 날짜를 계산해야 하는 번거로움
- 여러 기념일을 한 곳에 모아서 관리하고 싶은 니즈

## 핵심 기능

### 1. 기념일 등록

| 항목 | 설명 |
|------|------|
| 제목 | 기념일 이름 (예: "엄마 생신", "결혼기념일") |
| 날짜 | 양력 또는 음력 선택 후 날짜 입력 |
| 카운팅 방식 | D- (다가오는 날) 또는 D+ (경과일 카운팅) |
| 반복 | 매년 반복 여부 설정 |
| 메모 | 선물 아이디어, 장소 등 간단한 메모 (선택) |
| 카테고리 | 생일, 기념일, 공휴일 등 분류 (선택) |

#### 카운팅 방식에 따른 차이

| 방식 | 설명 | 예시 |
|------|------|------|
| **D- (다가오는 날)** | 다음 기념일까지 남은 일수 표시. 매년 반복과 함께 사용. | 생일, 기일, 결혼기념일, 설/추석 |
| **D+ (경과일)** | 시작일로부터 경과한 일수 표시. 100일/1000일 등 자동 계산. | 사귄 날, 입사일, 금연/금주 시작일 |

### 2. 기념일 목록 (홈 화면)

- 다가오는 기념일을 D-day 기준으로 정렬하여 보여줌
- 각 기념일 카드에 표시되는 정보:
  - 기념일 이름
  - D-day (예: D-15, D-Day, D+3)
  - 올해 양력 날짜 (음력인 경우 변환된 양력 날짜)
  - 음력 여부 표시

### 3. 음력 ↔ 양력 자동 변환

- 음력으로 등록한 기념일은 매년 해당하는 양력 날짜를 자동 계산
- 올해뿐 아니라 내년 날짜도 미리 확인 가능

### 4. 온보딩 — "어떤 기념일을 추가해볼까요?"

첫 실행 시 자주 쓰는 기념일 유형을 제안하여 빠르게 등록할 수 있도록 안내:

| 카테고리 | 제안 항목 |
|----------|-----------|
| 가족 | 부모님 생신(음력), 배우자 생일, 자녀 생일, 기일(제사) |
| 연인 | 사귄 날, 프로포즈 기념일, 결혼기념일 |
| 나 자신 | 입사일, 금연/금주/다이어트 시작일, 졸업일 |
| 기타 | 반려동물 생일/입양일, 이사한 날, 계약 만료일 |

- 제안 항목 선택 시 카운팅 방식(D-/D+), 음력 여부, 반복 설정이 자동으로 채워짐
- 건너뛰기 가능

### 5. 알림 (추후 구현)

- 기념일 당일, 1일 전, 3일 전, 7일 전 등 알림 설정
- 사용자가 알림 시점을 직접 선택 가능

## 데이터 모델 (초안)

```
Anniversary
├── id: Long
├── title: String           // 기념일 이름
├── date: LocalDate         // 원본 날짜 (양력 기준 저장)
├── isLunar: Boolean        // 음력 여부
├── lunarMonth: Int?        // 음력 월 (음력인 경우)
├── lunarDay: Int?          // 음력 일 (음력인 경우)
├── isLeapMonth: Boolean    // 윤달 여부
├── countingType: CountingType // D- 또는 D+ 방식
├── isRecurring: Boolean    // 매년 반복 여부
├── category: Category      // 카테고리
├── memo: String?           // 메모
├── createdAt: Instant      // 생성 시간
└── updatedAt: Instant      // 수정 시간

CountingType (enum)
├── D_MINUS    // 다가오는 날까지 카운트다운 (D-15, D-Day)
└── D_PLUS     // 시작일부터 경과일 카운팅 (D+100, D+1000)

Category (enum)
├── BIRTHDAY      // 생일
├── ANNIVERSARY   // 기념일
├── MEMORIAL      // 기일
├── HOLIDAY       // 공휴일
└── OTHER         // 기타
```

## 기술 고려사항

### 음력 변환

- 음력 ↔ 양력 변환 라이브러리 필요
- KMP에서 사용 가능한 방법 조사 필요 (한국천문연구원 API 또는 로컬 변환 라이브러리)

### 데이터 저장

- 로컬 DB 사용 (Room 또는 SQLDelight 등 KMP 호환 DB)
- 우선은 로컬 저장만 지원, 추후 클라우드 동기화 검토

### 플랫폼

- Android, iOS 동시 지원 (Compose Multiplatform)
